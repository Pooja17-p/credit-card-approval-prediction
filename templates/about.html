System Architecture (5-Tier Design)

Built with HTML5, Bootstrap 5, and customized Glassmorphic styling. This layer serves the dashboards, entry forms, flash prompts, and prediction result pages to the web client. Uses client-side JavaScript for validation and interactive visual animations.

Powered by Python and Flask. Handles routing configurations, analyst session mappings, CSRF validations, real-time input parsing, and executes the feature preprocessing pipeline (scaling and encoding) before requesting predictions.

Contains model training and prediction components. Uses scikit-learn standard scalers, one-hot categorical encoders, and a trained Random Forest model. Validates performance using classification reports (Accuracy, Recall, Precision, F1-Score, and ROC-AUC metrics) and confusion matrices.

Relational SQLite database managing 5 normalized tables (`Users`, `Applicant_Details`, `Credit_History`, `ML_Model`, and `Approval_Prediction`). Stores user credentials securely via SHA-256 hashes, logs applicant profiles, and logs prediction history results.

Connects local applications to remote IBM Cloud resources. The local Flask app is structured to delegate scaling operations to live API endpoints hosted on IBM Watson Machine Learning using deployment credentials.

Technology Stack Details

Frontend HTML5, CSS3 (Glassmorphism), JavaScript (ES6), Bootstrap 5, Bootstrap Icons.
Backend Controller Python, Flask, SQLite3, Joblib (Serialization).
Machine Learning Stack Pandas, NumPy, Scikit-learn, XGBoost, Matplotlib, Seaborn.
